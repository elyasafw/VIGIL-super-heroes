# VIGIL — ארכיון גיבורי־העל הסודי

שרת Backend שמנהל אוסף של רשומות גיבורי־על עבור סוכנות VIGIL. השרת שומר את הנתונים בקובץ JSON מקומי, וחושף API שמאפשר לסוכנים לחפש, לסנן, ליצור, לעדכן ולמחוק רשומות גיבורים, וכן לקבל תמונת מצב סטטיסטית על הארכיון.

## התקנה והרצה

```bash
npm install
node src/server.js
```

השרת עולה על פורט `3000`.

## מודל הנתונים — רשומת גיבור

| שדה            | הסבר                                                                      |
| ----------------- | ----------------------------------------------------------------------------- |
| `id`            | מזהה ייחודי של הרשומה                                       |
| `codeName`      | שם הקוד של הגיבור — שדה חובה, ייחודי              |
| `powers`        | מערך יכולות — שדה חובה, לפחות יכולת אחת        |
| `threatLevel`   | רמת איום/עוצמה — מספר שלם בין 1 ל־10                 |
| `status`        | סטטוס נוכחי —`active` / `retired` / `missing` / `deceased` |
| `origin`        | מוצא / רקע כללי של הגיבור                                  |
| `affiliations`  | מערך ארגונים/קבוצות שהגיבור משויך אליהן     |
| `firstSighting` | תאריך ההופעה הראשונה שתועדה                           |
| `notes`         | הערות חופשיות של סוכן VIGIL                                 |
| `createdAt`     | תאריך יצירת הרשומה — נקבע אוטומטית               |
| `updatedAt`     | תאריך העדכון האחרון — נקבע אוטומטית             |

## נתיבי API

| Method | נתיב           | תיאור                                                         |
| ------ | ------------------ | ------------------------------------------------------------------ |
| GET    | `/health`        | בדיקת תקינות לשרת                                   |
| GET    | `/heroes`        | רשימת גיבורים, עם סינון / מיון / pagination |
| GET    | `/heroes/:id`    | שליפת גיבור בודד לפי מזהה                     |
| POST   | `/heroes`        | יצירת רשומת גיבור חדשה                          |
| PATCH  | `/heroes/:id`    | עדכון רשומה קיימת (עדכון חלקי)             |
| DELETE | `/heroes/:id`    | מחיקת רשומה                                              |
| POST   | `/heroes/search` | חיפוש מתקדם עם מספר תנאים דרך body         |
| GET    | `/stats`         | סטטיסטיקות מצטברות על הארכיון            |

כל נתיב שלא מוגדר ברשימה מחזיר `404`.

### פרמטרים לסינון/מיון/pagination ב־`GET /heroes`

| פרמטר   | הסבר                                                                      |
| ------------ | ----------------------------------------------------------------------------- |
| `status`   | מחזיר רק גיבורים בסטטוס הנתון                        |
| `power`    | מחזיר גיבורים עם היכולת הנתונה במערך`powers` |
| `minLevel` | `threatLevel` גדול או שווה לערך                               |
| `maxLevel` | `threatLevel` קטן או שווה לערך                                 |
| `search`   | חיפוש טקסט חופשי בתוך`codeName` או `notes`            |
| `sortBy`   | מיון לפי שדה (`codeName` / `threatLevel` / `firstSighting`)   |
| `order`    | כיוון המיון —`asc` (ברירת מחדל) או `desc`           |
| `page`     | מספר עמוד (ברירת מחדל: 1)                                    |
| `limit`    | תוצאות בעמוד (ברירת מחדל: 20)                             |

### חיפוש מתקדם — `POST /heroes/search`

מקבל בגוף הבקשה שילוב של `statuses`, `powers`, `minLevel`, `maxLevel`, `affiliations`. כל השדות אופציונליים ומחוברים ב־ AND;

body ריק מחזיר את כל הגיבורים.

### סטטיסטיקות — `GET /stats`

מחזיר `totalHeroes`, `byStatus`, `averageThreatLevel`, `mostCommonPower`, `highestThreat`, `newestRecord`.

## מבנה תשובות וקודי סטטוס

הצלחה:

```json
{ "success": true, "data": {} }
```

שגיאה:

```json
{ "success": false, "message": "Error message" }
```

| קוד | מתי                                                |
| ------ | ----------------------------------------------------- |
| 200    | קריאה, עדכון או מחיקה שהצליחו |
| 201    | יצירת גיבור חדש                          |
| 400    | בקשה לא תקינה (ולידציה נכשלה)  |
| 404    | גיבור או נתיב שלא נמצאו            |
| 409    | כפילות —`codeName` שכבר קיים         |
| 500    | שגיאה פנימית בשרת                      |

## מבנה קוד

```
controllers/   # טיפול בבקשה ובתשובה
routes/        # הגדרת הנתיבים
services/      # לוגיקה: סינון, מיון, סטטיסטיקות
utils/         # ולידציה ומודול תשובות אחיד
src/           # קובץ הפעלת השרת
```
